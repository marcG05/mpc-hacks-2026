# 🎯 Fraud Hunter — Straight Forward Build Plan

## Tech Stack

| Layer | Choice | Reason |
|-------|--------|--------|
| **Frontend** | Next.js 15 + Tailwind + shadcn/ui | Fast to build, keyboard events are trivial, great table/card components |
| **Backend** | Python + FastAPI | Pandas for detection, clean REST API, easy MCP integration |
| **Database** | SQLite | Zero config, stores decisions + audit log, ships in the repo |
| **AI / Chatbot** | Claude API (claude-sonnet-4-5) | Human-readable explanations + chatbot |
| **MCP Server** | Python MCP SDK | Expose transaction tools to Claude |
| **Charts** | Recharts | Built into React, zero config |

## Detection Engine — What It Actually Computes

```python
# For every transaction, build a composite score from these signals:

SIGNALS = {
    # Per-card behavioral anomalies
    "amount_zscore":         weight=0.25,  # How many σ above card's median
    "new_category":          weight=0.15,  # Category never seen on this card
    "new_channel":           weight=0.10,  # e.g. card always in-person, now online
    "unusual_hour":          weight=0.10,  # Outside card's typical active hours

    # Cross-card signals
    "device_reuse":          weight=0.20,  # device_id seen on N other cards
    "ip_reuse":              weight=0.20,  # ip_address seen on N other cards

    # Geographic
    "country_mismatch":      weight=0.15,  # cardholder_country ≠ merchant_country

    # Velocity
    "velocity_1h":           weight=0.15,  # N+ txns from card in 1 hour
    "card_testing_pattern":  weight=0.20,  # Small probe followed by large txn
}

# Final score = weighted sum, clipped to [0, 1]
# Threshold tuned to maximize F1 on the dataset
```

**Human-readable reasons** → Claude receives the triggered signals and writes a 2–3 sentence natural language explanation per transaction.
